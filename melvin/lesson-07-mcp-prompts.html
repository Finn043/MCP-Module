1. Prompts là gì?

Prompts là primitive được điều khiển bởi người dùng (user-controlled). Người dùng trigger prompt thông qua slash commands, buttons, hoặc menu selections.

Đặc điểm chính của Prompts:

  • Được điều khiển bởi người dùng (user)
  • Là pre-built instruction templates — đã được test và tối ưu
  • Encode domain expertise — kiến thức chuyên môn được đóng gói
  • User trigger qua slash commands (/format), buttons, hoặc menu
Hãy tưởng tượng bạn đến nhà hàng. Bạn có thể tự nấu ăn (ad-hoc instructions), nhưng món ăn do đầu bếp chuẩn bị (prompts) luôn ngon hơn vì đầu bếp đã dành nhiều thời gian hoàn thiện công thức. MCP Prompts chính là những "công thức" được đầu bếp (server author) cẩn thận chuẩn bị.

2. Tại sao Prompts cho kết quả tốt hơn?

Người dùng có thể hỏi Claude trực tiếp, nhưng prompts mang lại kết quả nhất quán và chất lượng hơn.

Lợi ích của Prompts so với ad-hoc instructions:

  • Pre-crafted — Được soạn sẵn cẩn thận, không phải tự viết
  • Tested — Đã thử nghiệm qua nhiều scenarios
  • Domain-specific — Tối ưu cho từng lĩnh vực cụ thể
  • Consistent — Kết quả nhất quán mỗi lần sử dụng
  • Maintainable — Cập nhật một nơi, cải thiện tất cả clients

Ví dụ: User có thể gõ "reformat report.pdf thành markdown" và nhận kết quả khá. Nhưng prompt /format được server author viết, test và tối ưu sẽ cho kết quả tốt hơn nhiều — xử lý edge cases, follow best practices.

3. Định nghĩa Prompt

Sử dụng @mcp.prompt() decorator. Function trả về list[base.Message].

from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base
from pydantic import Field

mcp = FastMCP("DocumentMCP")

@mcp.prompt(
    name="format",
    description="Rewrites the contents of the document in Markdown format.",
)
def format_document(
    doc_id: str = Field(description="Id of the document to format"),
) -> list[base.Message]:
    prompt = f"""
    Your goal is to reformat a document to be written with markdown syntax.

    The id of the document you need to reformat is:
    <document_id>
    {doc_id}
    </document_id>

    Add in headers, bullet points, tables, etc as necessary. Feel free to add
    in extra text, but don't change the meaning of the report.
    Use the 'edit_document' tool to edit the document.
    After the document has been edited, respond with the final version of the doc.
    Don't explain your changes.
    """
    return [base.UserMessage(prompt)]

Các thành phần quan trọng:

  • @mcp.prompt() — Decorator đăng ký prompt với server
  • name — Tên prompt (dùng trong slash command: /format)
  • description — Mô tả prompt (hiển thị cho client)
  • Field() — Mô tả parameter, giúp client biết cần input gì
  • list[base.Message] — Trả về danh sách messages
  • base.UserMessage() — Tạo user message (có thể có nhiều messages)

4. prompts/list và prompts/get

prompts/list — Liệt kê tất cả prompts có sẵn

// Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "prompts/list"
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "prompts": [
      {
        "name": "format",
        "description": "Rewrites the contents of the document in Markdown format.",
        "arguments": [
          {
            "name": "doc_id",
            "description": "Id of the document to format",
            "required": true
          }
        ]
      }
    ]
  }
}

prompts/get — Lấy prompt cụ thể với arguments

// Request
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "prompts/get",
  "params": {
    "name": "format",
    "arguments": {
      "doc_id": "plan.md"
    }
  }
}

// Response
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "description": "Rewrites the contents of the document in Markdown format.",
    "messages": [
      {
        "role": "user",
        "content": {
          "type": "text",
          "text": "Your goal is to reformat a document to be written with markdown syntax.\n\nThe id of the document you need to reformat is:\n<document_id>\nplan.md\n</document_id>\n\nAdd in headers, bullet points, tables, etc..."
        }
      }
    ]
  }
}

5. Prompt Lifecycle

Vòng đời hoàn chỉnh của một prompt từ lúc tạo đến khi gửi cho Claude.

1. Viết prompt → 2. Định nghĩa với decorator → 3. Client request

→ 4. Arguments interpolated → 5. Messages returned → 6. Gửi cho Claude
1
Viết và đánh giá — Server author soạn thảo prompt, test với nhiều scenarios, tinh chỉnh cho kết quả tốt nhất.
2
Định nghĩa với decorator — Dùng @mcp.prompt() để đăng ký prompt với MCP server.
3
Client request — Client gọi prompts/list để khám phá, hoặc gọi trực tiếp prompts/get.
4
Arguments interpolated — Arguments từ client (vd: doc_id="plan.md") được chèn vào prompt template.
5
Messages returned — Function trả về list[Message] đã được điền đầy đủ thông tin.
6
Gửi cho Claude — Messages được convert và gửi cho Claude như một phần của conversation.

6. Slash Command Pattern

Trong CLI app, user gõ slash command để trigger prompt.

# Xử lý slash command trong CLI
async def _process_command(self, query: str) -> bool:
    if not query.startswith("/"):
        return False

    words = query.split()
    command = words[0].replace("/", "")  # "/format" → "format"

    # Gọi prompts/get với arguments
    messages = await self.doc_client.get_prompt(
        command,           # prompt name
        {"doc_id": words[1]}  # arguments
    )

    # Convert và thêm vào conversation
    self.messages += convert_prompt_messages_to_message_params(messages)
    return True

Workflow khi user gõ /format plan.md:

  1. App nhận diện / → đây là slash command
  2. Extract: command = "format", argument = "plan.md"
  3. Gọi get_prompt("format", {"doc_id": "plan.md"})
  4. Server trả về messages với plan.md đã được interpolate
  5. Messages được inject vào conversation → gửi cho Claude
  6. Claude sử dụng tools để đọc và format document
