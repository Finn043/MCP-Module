1. Tại sao cần Initialize?

MCP yêu cầu một quá trình handshake (bắt tay) bắt buộc trước khi gửi bất kỳ request nào. Handshake đảm bảo cả client và server hiểu nhau về phiên bản giao thức và khả năng hỗ trợ.

Giống như khi gọi điện thoại: bạn phải "Alo" trước, xác nhận người nghe đúng số, rồi mới bắt đầu nói chuyện. Trong MCP, handshake chính là tiếng "Alo" đó — ba bước để thiết lập kết nối trước khi làm việc thực sự.

2. Ba bước Handshake

Client -- Initialize Request --> Server

Client <-- Initialize Result -- Server

Client -- Initialized Notification --> Server
(Khong co response)
1
Initialize Request — Client gửi yêu cầu khởi tạo, kèm theo protocolVersion mà client hỗ trợ và clientCapabilities (những gì client có thể làm).
2
Initialize Result — Server phản hồi với protocolVersion được chốt, serverCapabilities (những gì server hỗ trợ), và thông tin server.
3
Initialized Notification — Client gửi notification xác nhận đã nhận kết quả. Đây là message một chiều — không có response.

3. Initialize Request — Chi tiết

Client gửi JSON-RPC request với method "initialize":

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2024-11-05",
    "capabilities": {
      "sampling": {},
      "roots": { "listChanged": true }
    },
    "clientInfo": {
      "name": "my-mcp-client",
      "version": "1.0.0"
    }
  }
}

clientCapabilities có thể bao gồm:

CapabilityMô tả
samplingClient có thể gọi LLM thay server (sampling callback)
rootsClient có thể cung cấp roots (thư mục/truy cập file)

4. Initialize Result — Chi tiết

Server phản hồi với capabilities của mình:

{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2024-11-05",
    "capabilities": {
      "tools": { "listChanged": true },
      "resources": { "subscribe": true, "listChanged": true },
      "prompts": { "listChanged": true },
      "logging": {}
    },
    "serverInfo": {
      "name": "github-mcp-server",
      "version": "2.1.0"
    }
  }
}

serverCapabilities có thể bao gồm:

CapabilityMô tả
toolsServer cung cấp tools cho Claude gọi
resourcesServer cung cấp resources cho app fetch
promptsServer cung cấp prompt templates
loggingServer có thể gửi log messages

5. Initialized Notification — Chi tiết

Sau khi nhận Initialize Result, client gửi notification xác nhận:

{
  "jsonrpc": "2.0",
  "method": "notifications/initialized"
}

Đặc điểm quan trọng:

  • Không có trường id — Đây là notification, không mong đợi response
  • Không có params — Notification đơn giản, chỉ xác nhận "tôi đã sẵn sàng"
  • Server KHÔNG trả về response — Một chiều, fire-and-forget

6. Sau Handshake

Chỉ SAU khi cả 3 bước hoàn tất mới có thể gửi các requests khác:

  • tools/list — Liệt kê tools
  • tools/call — Gọi một tool
  • resources/list — Liệt kê resources
  • resources/read — Đọc một resource
  • prompts/list — Liệt kê prompts
  • prompts/get — Lấy một prompt

Nếu gửi bất kỳ request nào trước khi initialize, server sẽ trả về error.

Session ID trong HTTP Transport

Khi sử dụng StreamableHTTP transport, server trả thêm mcp-session-id trong response header của Initialize Result. Session ID này phải được gửi kèm trong mọi request sau đó để server nhận diện client. Nếu dùng stdio transport, không cần session ID vì client và server chạy trên cùng một tiến trình.

Mo phong Handshake MCP

Thuc hien 3 buoc handshake de ket noi MCP Client voi MCP Server. Chon capabilities de thay doi noi dung JSON.

Client Capabilities

Server Capabilities

Buoc 1: Initialize Request
Cho

Client gui yeu cau khoi tao voi protocolVersion va clientCapabilities.

Buoc 2: Initialize Result
Cho

Server phan hoi voi serverCapabilities va protocolVersion.

Buoc 3: Initialized Notification
Cho
