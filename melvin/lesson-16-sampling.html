1. Sampling là gì?

Trong kiến trúc MCP bình thường, client là bên gọi AI model (Claude) và server cung cấp tools/resources. Nhưng đôi khi server cũng cần khả năng của LLM — ví dụ: tóm tắt một đoạn text dài, phân tích dữ liệu phức tạp, hay tạo nội dung.

Thay vì tự gọi API (cần API key, xử lý authentication, chịu phí token), server có thể nhờ client gọi Claude thay mình. Client đã có sẵn connection đến Claude, nên đây là cách tự nhiên và hiệu quả nhất.

Bạn đến nhà hàng và muốn một ly sinh tố. Thay vì tự mua máy xay và nguyên liệu (đắt tiền, phức tạp), bạn nhờ người phục vụ mang ra cho mình. Người phục vụ đã có sẵn bếp và nguyên liệu — bạn chỉ cần đặt hàng và trả tiền.

2. Hai cách tiếp cận

Khi server cần gọi LLM, có hai lựa chọn:

CáchCách tiếp cậnTrade-off
Direct API AccessServer tự gọi API với API key riêng, xử lý auth và costThêm complexity đáng kể cho server
SamplingServer tạo prompt, nhờ client gọi Claude thayServer giữ đơn giản; client chịu cost và auth
🔑 Điểm chính: Sampling "đảo ngược" flow thông thường. Thay vì client hỏi server, server chủ động gửi request đến client. Đây là ví dụ điển hình của giao thức bidirectional trong MCP.

3. Server-side Implementation

Trên server, bạn dùng ctx.session.create_message() để gửi sampling request đến client. Method này nhận các tham số:

  • messages — Danh sách SamplingMessage với role và content
  • max_tokens — Giới hạn tokens cho response
  • system_prompt — Hướng dẫn hệ thống cho LLM (optional)
# server.py
from mcp.server.fastmcp import FastMCP, Context
from mcp.types import SamplingMessage, TextContent

mcp = FastMCP(name="Demo Server")

@mcp.tool()
async def summarize(text_to_summarize: str, ctx: Context):
    prompt = f"""
        Please summarize the following text:
        {text_to_summarize}
    """

    result = await ctx.session.create_message(
        messages=[
            SamplingMessage(
                role="user",
                content=TextContent(type="text", text=prompt)
            )
        ],
        max_tokens=4000,
        system_prompt="You are a helpful research assistant.",
    )

    if result.content.type == "text":
        return result.content.text
    else:
        raise ValueError("Sampling failed")

4. Client-side Implementation

Client cần cung cấp sampling_callback cho ClientSession. Callback này:

  1. Nhận messages từ server (định dạng MCP)
  2. Convert sang định dạng tương thích với Anthropic SDK
  3. Gọi Claude và trả về CreateMessageResult
# client.py
from anthropic import AsyncAnthropic
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client
from mcp.client.session import RequestContext
from mcp.types import (
    CreateMessageRequestParams, CreateMessageResult,
    TextContent, SamplingMessage,
)

anthropic_client = AsyncAnthropic()
model = "claude-sonnet-4-0"

async def chat(input_messages: list[SamplingMessage], max_tokens=4000):
    """Convert SamplingMessages to Anthropic SDK format."""
    messages = []
    for msg in input_messages:
        if msg.role == "user" and msg.content.type == "text":
            content = (
                msg.content.text
                if hasattr(msg.content, "text")
                else str(msg.content)
            )
            messages.append({"role": "user", "content": content})
        elif msg.role == "assistant" and msg.content.type == "text":
            content = (
                msg.content.text
                if hasattr(msg.content, "text")
                else str(msg.content)
            )
            messages.append({"role": "assistant", "content": content})

    response = await anthropic_client.messages.create(
        model=model,
        messages=messages,
        max_tokens=max_tokens,
    )
    text = "".join([p.text for p in response.content if p.type == "text"])
    return text

async def sampling_callback(
    context: RequestContext, params: CreateMessageRequestParams
):
    text = await chat(params.messages)
    return CreateMessageResult(
        role="assistant",
        model=model,
        content=TextContent(type="text", text=text),
    )

async def run():
    async with stdio_client(server_params) as (read, write):
        async with ClientSession(
            read, write, sampling_callback=sampling_callback
        ) as session:
            await session.initialize()
            result = await session.call_tool(
                name="summarize",
                arguments={"text_to_summarize": "lots of text"},
            )
            print(result.content)

5. 6-Step Walkthrough

1
Server gọi create_message() — Trong lúc thực thi tool, server gọi ctx.session.create_message() với prompt cần tóm tắt.
2
Client nhận sampling request — MCP framework tự động chuyển request đến sampling_callback trên client.
3
Callback convert messages — Callback chuyển đổi SamplingMessage objects sang định dạng Anthropic SDK (dict với role/content).
4
Client gọi Claude — Sử dụng Anthropic SDK, client gửi prompt đến Claude và nhận response.
5
Trả kết quả về server — Client wrap kết quả trong CreateMessageResult và gửi ngược về server.
6
Server sử dụng kết quả — Server nhận text đã generate và trả về cho người dùng.
Server Tool → create_message() → Client Callback → convert → Claude API → response → Server Tool

6. Khi nào dùng Sampling?

Sử dụng Sampling khi:
  • Xây dựng public MCP servers — không muốn absorb AI costs cho mỗi user
  • Server cần LLM nhưng không muốn quản lý API keys
  • Client đã có connection đến Claude — tận dụng thay vì tạo mới
  • Muốn shift authentication burden đến client
Không dùng Sampling khi:
  • Server chạy nội bộ (private) — có thể dùng API key trực tiếp
  • Dùng stateless_http=True — sampling không hoạt động
  • Không cần LLM capabilities trong server
