1. Tổng quan 3 Primitives

Mỗi primitive được điều khiển bởi một phần khác nhau trong ứng dụng của bạn. Hiểu rõ sự khác biệt là chìa khóa để thiết kế MCP server tốt.

2. Decision Guide

Hỏi mình 3 câu hỏi đơn giản để chọn đúng primitive:

Câu hỏi 1: Bạn cần Claude tự động thực hiện hành động (không cần user nhắc)?
→ Dùng TOOLS (model-controlled)
Câu hỏi 2: Bạn cần fetch data vào ứng dụng (cho UI hoặc augment prompt)?
→ Dùng RESOURCES (app-controlled)
Câu hỏi 3: Bạn muốn cung cấp predefined workflows cho user trigger?
→ Dùng PROMPTS (user-controlled)

3. Real-world Examples từ Claude's Interface

Nhìn vào giao diện Claude để hiểu rõ hơn cách 3 primitives được sử dụng.

Tools trong Claude:

Khi Claude tự động thực thi code, tìm kiếm web, hoặc thực hiện tính toán — đó là Tools. Claude quyết định khi nào cần dùng, không cần bạn yêu cầu.

Ví dụ: Bạn hỏi "Tính căn bậc hai của 144" → Claude tự quyết định dùng code execution tool.

Resources trong Claude:

Tính năng "Add from Google Drive" — ứng dụng fetch nội dung file và inject vào conversation. Không phải Claude gọi, mà app quyết định.

Ví dụ: Bạn chọn file từ Google Drive → App đọc nội dung → Inject vào prompt → Claude có context ngay.

Prompts trong Claude:

Các workflow buttons bên dưới chat input — predefined, optimized workflows. User click → trigger prompt → Claude thực hiện workflow đã được chuẩn bị.

Ví dụ: Click "Summarize this document" → Prompt template được gửi cho Claude.

4. Lỗi Thường Gặp

Lỗi phổ biến nhất: Dùng Tools khi Resources phù hợp hơn.

Nhiều developers định nghĩa tools để đọc data (như read_document), trong khi Resources sẽ hiệu quả hơn cho read-only operations.

Tại sao Resources tốt hơn cho read-only data?

Tiêu chíDùng ToolDùng Resource
Số lần gọi API2+ lần (Claude hỏi → tool gọi → kết quả)1 lần (inject trực tiếp vào prompt)

LatencyChậm hơnNhanh hơn
Claude's contextPhải đợi tool result mới có dataCó data ngay từ prompt đầu tiên

Nguyên tắc vàng:

  • Nếu chỉ đọc data → dùng Resource
  • Nếu cần thực hiện hành động (ghi, sửa, xóa, gọi API) → dùng Tool
  • Nếu cần pre-defined workflow cho user → dùng Prompt

5. Ba Primitives Hoạt Động Cùng Nhau

Trong thực tế, một MCP Server sử dụng tất cả ba primitives cùng lúc. Mỗi primitive phục vụ một mục đích khác nhau.

Ví dụ: Document Manager MCP Server

Tools (model-controlled)

@mcp.tool(name="read_doc_contents", ...)
def read_document(doc_id: str): ...

@mcp.tool(name="edit_document", ...)
def edit_document(doc_id: str, old_str: str, new_str: str): ...

Claude tự quyết định khi nào đọc hoặc sửa document dựa trên user request.

Resources (app-controlled)

@mcp.resource("docs://documents", mime_type="application/json")
def list_docs() -> list[str]: ...

@mcp.resource("docs://documents/{doc_id}", mime_type="text/plain")
def fetch_doc(doc_id: str) -> str: ...

App fetch danh sách docs cho autocomplete, hoặc inject nội dung khi user gõ @report.pdf.

Prompts (user-controlled)

@mcp.prompt(name="format", ...)
def format_document(doc_id: str) -> list[base.Message]: ...

User gõ /format plan.md → Prompt được trigger → Claude format document.

Document Manager — 3 Primitives hoạt động cùng nhau

read_doc_contents (Tool) edit_document (Tool)
↑ Claude tự gọi khi cần

docs://documents (Resource) docs://documents/{id} (Resource)
↑ App fetch khi cần (autocomplete, @mention)

/format (Prompt)
↑ User trigger qua slash command
