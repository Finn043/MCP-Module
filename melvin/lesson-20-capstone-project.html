3. Complete Flow Walkthrough

10 bước diễn ra khi user tương tác với ứng dụng:

1
User chạy main.py với root paths — uv run main.py /Users/me/Desktop. Client khởi tạo với roots được chỉ định.
2
Client spawns server subprocess — stdio transport start, server process launched.
3
Initialize handshake (3 messages) — Initialize Request → Initialize Result → Initialized Notification. Capabilities exchanged.
4
User types query — "convert biking.mp4 to mov". Client sends query to Claude.
5
Claude calls tools/list — Discovers available tools, understands what it can do.
6
Claude calls tool (với progress notifications) — convert_video() executes. Server sends ctx.info() và ctx.report_progress() cho real-time feedback.
7
Roots check — Server calls ctx.session.list_roots() và is_path_allowed() để verify file access.
8
Resource fetched via @mention — Nếu user mention một resource, client fetches qua read_resource().
9
Sampling for summarization — Nếu tool cần tóm tắt, server calls ctx.session.create_message() → client callback → Claude → result trả về.
10
Result displayed — Client hiển thị kết quả cho user. Tool execution complete.

4. Production Deployment

Khi chuyển từ development sang production:

AspectDevelopmentProduction
TransportstdioStreamableHTTP
ModeDefault (stateful)Stateful hoặc Stateless tùy scaling needs
Error handlingBasic print statementsStructured error responses, retry logic
TestingManualAutomated với production transport
Checklist trước khi deploy:
  • Chọn transport phù hợp (stdio cho local, HTTP cho remote)
  • Quyết định stateful vs stateless dựa trên scaling needs
  • Test sampling, notifications, roots với production transport
  • Implement error handling cho transport errors
  • Verify security — is_path_allowed() cho mọi file access

Complete Flow Simulator — Grand Finale

Nhấn "Start" để xem toàn bộ flow hoạt động, từ user input đến final result. Mỗi bước có thể click để xem JSON-RPC message bên dưới.

Sẵn sàng

1 User Input — User types: "convert biking.mp4 to mov" 
// User input
> convert biking.mp4 to mov

// CLI sends to Claude API
messages: [
  {"role": "user", "content": "convert biking.mp4 to mov"}
]
2 Initialize Handshake — 3-message MCP handshake 
// 1. Initialize Request (Client → Server)
{"jsonrpc":"2.0","method":"initialize","id":1,
  "params":{"protocolVersion":"2024-11-05",
    "capabilities":{},
    "clientInfo":{"name":"DocManager","version":"1.0"}}}

// 2. Initialize Result (Server → Client)
{"jsonrpc":"2.0","id":1,
  "result":{"protocolVersion":"2024-11-05",
    "capabilities":{"tools":{},"resources":{}},
    "serverInfo":{"name":"VidsMCP"}}}

// 3. Initialized Notification (Client → Server)
{"jsonrpc":"2.0","method":"notifications/initialized"}
3 List Tools — Claude discovers available tools 
// Request
{"jsonrpc":"2.0","method":"tools/list","id":2}

// Response
{"jsonrpc":"2.0","id":2,"result":{
  "tools":[
    {"name":"convert_video",
     "description":"Convert MP4 to another format",
     "inputSchema":{
       "type":"object",
       "properties":{
         "input_path":{"type":"string"},
         "format":{"type":"string"}}}},
    {"name":"list_roots","description":"List accessible dirs"},
    {"name":"read_dir","description":"Read directory contents"}
  ]
}}
4 Roots Discovery — Server discovers accessible directories 
// Server → Client: roots/list
{"jsonrpc":"2.0","method":"roots/list","id":3}

// Client → Server: roots response
{"jsonrpc":"2.0","id":3,"result":{
  "roots":[
    {"uri":"file:///Users/melvin/Desktop","name":"Desktop"},
    {"uri":"file:///Users/melvin/Movies","name":"Movies"}
  ]
}}
5 Tool Call + Progress Notifications — convert_video executes 
// Claude calls tool
{"jsonrpc":"2.0","method":"tools/call","id":4,
  "params":{"name":"convert_video",
    "arguments":{"input_path":"/Users/melvin/Desktop/biking.mp4",
                 "format":"mov"},
    "_meta":{"progressToken":"prog-1"}}}

// Server sends progress notifications
{"jsonrpc":"2.0","method":"notifications/progress",
  "params":{"progressToken":"prog-1",
    "progress":20,"total":100}}
{"jsonrpc":"2.0","method":"notifications/progress",
  "params":{"progressToken":"prog-1",
    "progress":80,"total":100}}

// Final result
{"jsonrpc":"2.0","id":4,"result":{
  "content":[{"type":"text",
    "text":"Successfully converted biking.mp4 to biking.mov"}]}}
6 Sampling (if needed) — Server requests Claude for summarization 
// Server → Client: sampling request
{"jsonrpc":"2.0","method":"sampling/createMessage","id":5,
  "params":{
    "messages":[{"role":"user",
      "content":{"type":"text",
        "text":"Summarize: biking.mp4 (45MB, 1080p)"}}],
    "maxTokens":500,
    "systemPrompt":"You are a helpful assistant."}}

// Client calls Claude, returns result
{"jsonrpc":"2.0","id":5,"result":{
  "role":"assistant",
  "model":"claude-sonnet-4-0",
  "content":{"type":"text",
    "text":"Video về đạp xe, chất lượng 1080p..."}}}}
7 Final Result — User sees the completed result 
// CLI output to user
✅ Conversion complete!
   Input:  /Users/melvin/Desktop/biking.mp4
   Output: /Users/melvin/Desktop/biking.mov
   Size:   32MB (compressed from 45MB)

Summary: Video về đạp xe, chất lượng 1080p...
