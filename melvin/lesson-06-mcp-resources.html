Bài 06: MCP Resources — Đưa dữ liệu vào ứng dụng

Resources là endpoints đọc dữ liệu — app-controlled, hiệu quả hơn Tools cho read-only data

1. Resources là gì?

Resources là primitive được điều khiển bởi ứng dụng (app-controlled). Ứng dụng của bạn quyết định khi nào fetch data — không phải Claude, cũng không phải người dùng.

Đặc điểm chính của Resources:

  • Được điều khiển bởi ứng dụng (app code)
  • Giống như GET endpoints trong HTTP — chỉ đọc data
  • Data được inject trực tiếp vào prompt — không cần tool call round-trip
  • Hiệu quả hơn Tools cho read-only data
Nếu Tools là "hành động" (POST/PUT), thì Resources là "đọc dữ liệu" (GET). Trong giao diện Claude, tính năng "Add from Google Drive" chính là Resource — ứng dụng fetch nội dung file và inject vào conversation.

2. Direct Resources vs Templated Resources

Direct Resources (Static URI)

URI cố định, trả về cùng một loại data mỗi khi gọi.

@mcp.resource("docs://documents", mime_type="application/json")
def list_docs() -> list[str]:
    """Trả về danh sách tất cả tài liệu."""
    return list(docs.keys())

Templated Resources (Parameterized URI)

URI chứa tham số trong ngoặc nhọn {param}, cho phép fetch data cụ thể.

@mcp.resource("docs://documents/{doc_id}", mime_type="text/plain")
def read_doc(doc_id: str) -> str:
    """Trả về nội dung tài liệu theo ID."""
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]

So sánh:

3. MIME Types

Server chỉ định MIME type cho mỗi resource, giúp client biết cách xử lý data.

MIME TypeDùng choVí dụ
application/jsonDữ liệu có cấu trúcDanh sách documents, metadata
text/plainVăn bản thông thườngNội dung tài liệu
application/pdfFile PDFBáo cáo PDF

SDK xử lý serialization tự động — bạn chỉ cần return data, SDK lo phần còn lại.

  • Resource trả về list hoặc dict với mime_type="application/json" sẽ được tự động chuyển thành JSON string
  • Resource trả về str với mime_type="text/plain" sẽ giữ nguyên

4. resources/list và resources/read

resources/list — Liệt kê tất cả resources

// Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "resources/list"
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "resources": [
      {
        "uri": "docs://documents",
        "name": "list_docs",
        "mimeType": "application/json"
      }
    ],
    "resourceTemplates": [
      {
        "uriTemplate": "docs://documents/{doc_id}",
        "name": "read_doc",
        "mimeType": "text/plain"
      }
    ]
  }
}

resources/read — Đọc nội dung resource theo URI

// Request
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "resources/read",
  "params": {
    "uri": "docs://documents/report.pdf"
  }
}

// Response
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "contents": [
      {
        "uri": "docs://documents/report.pdf",
        "mimeType": "text/plain",
        "text": "The report details the state of a 20m condenser tower."
      }
    ]
  }
}

5. Tại sao Resources hiệu quả hơn Tools cho read-only data?

Cách 1: Dùng Tool (chậm hơn)
User hỏi về document → Claude quyết định gọi tool → tools/call round-trip → Claude nhận kết quả → Trả lời user

Cách 2: Dùng Resource (nhanh hơn)
User hỏi về document → App fetch resource trực tiếp → Inject vào prompt → Claude trả lời ngay

Lợi ích của Resources so với Tools cho read-only data:

  • Không cần tool call round-trip — nội dung được inject trực tiếp vào prompt ban đầu
  • Tiết kiệm tokens — không cần Claude hỏi rồi mới lấy data
  • Giảm latency — user nhận được câu trả lời nhanh hơn vì không cần vòng lặp tool use
  • Claude có context ngay từ đầu — trả lời chính xác hơn

6. @mention Pattern

Trong CLI apps, người dùng có thể gõ @ để reference một resource. Ứng dụng tự động fetch nội dung và inject vào prompt.

# Trong CLI app
async def _extract_resources(self, query: str) -> str:
    """Trích xuất @mentions từ query và fetch nội dung."""
    mentions = [word[1:] for word in query.split() if word.startswith("@")]
    doc_ids = await self.list_docs_ids()
    mentioned_docs = []

    for doc_id in doc_ids:
        if doc_id in mentions:
            content = await self.get_doc_content(doc_id)
            mentioned_docs.append((doc_id, content))

    return "".join(
        f'\n<document id="{doc_id}">\n{content}\n</document>\n'
        for doc_id, content in mentioned_docs
    )

Ví dụ workflow:

  1. User gõ: "Hãy tóm tắt @report.pdf"
  2. App nhận diện @report.pdf là resource reference
  3. App gửi resources/read với URI docs://documents/report.pdf
  4. App inject nội dung vào prompt: <document id="report.pdf">...</document>
  5. Claude nhận được prompt kèm nội dung document → trả lời ngay lập tức
