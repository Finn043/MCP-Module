1. Project Setup -- Cau hinh du an

Truoc khi bat dau xay dung MCP Server, chung ta can thiet lap du an voi cac dependencies can thiet. Du an nay la mot CLI chat ung dung quan ly tai lieu su dung MCP.

Dependencies

# pyproject.toml
dependencies = [
    "anthropic>=0.51.0",
    "mcp[cli]>=1.8.0",
    "prompt-toolkit>=3.0.51",
    "python-dotenv>=1.1.0",
]

mcp[cli]>=1.8.0 -- MCP Python SDK voi CLI tools (bao gom Inspector). anthropic>=0.51.0 -- Anthropic Python SDK de goi Claude.

Cau truc thu muc du an

cli_project/
├── .env                    # API keys va config
├── main.py                 # Entry point -- ket noi moi thu
├── mcp_server.py           # MCP server voi tools, resources, prompts
├── mcp_client.py           # MCP client -- cau noi den MCP server
├── pyproject.toml          # Dependencies
├── core/
│   ├── __init__.py
│   ├── claude.py           # Claude API wrapper
│   ├── chat.py             # Base chat logic voi tool-use loop

│   ├── cli.py              # CLI interface (autocomplete, keybindings)
│   └── tools.py            # Tool discovery va execution manager
Nghep cua MCP Server nhu mot "nha hang" -- ban dinh nghia menu (tools, resources, prompts) va khach hang (MCP Client) co the goi mon qua giao thuc chuan hoa. Khong can biet bep hoat dong nhu the nao, chi can biet cach goi mon.

2. FastMCP Initialization -- Khoi tao Server

FastMCP la lop chinh trong Python MCP SDK. Nó cho phep ban dinh nghia server chi bang decorator va type hints -- khong can viet JSON Schema thu cong.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("DocumentMCP", log_level="ERROR")

docs = {
    "deposition.md": "This deposition covers the testimony of Angela Smith, P.E.",
    "report.pdf": "The report details the state of a 20m condenser tower.",
    "financials.docx": "These financials outline the project's budget and expenditures.",
    "outlook.pdf": "This document presents the projected future performance of the system.",
    "plan.md": "The plan outlines the steps for the project's implementation.",
    "spec.txt": "These specifications define the technical requirements for the equipment.",
}

Giai thich

FastMCP("DocumentMCP") -- Khoi tao MCP Server voi ten "DocumentMCP". Ten nay xuat hien khi client discovery.

log_level="ERROR" -- Chi hien thi loi nghiêm trong. Giup terminal sach hon khi development.

docs -- Dictionary luu tru tai lieu trong memory. Moi key la ten file, value la noi dung.

3. Dinh nghia Tools -- Model-Controlled Primitives

Tools la nhung ham ma Claude tu quyet dinh khi goi. SDK tu dong chuyen doi Python function thanh JSON Schema cho tool definition.

Tool 1: read_doc_contents

from pydantic import Field

@mcp.tool(
    name="read_doc_contents",
    description="Read the contents of a document and return it as a string.",
)
def read_document(
    doc_id: str = Field(description="Id of the document to read"),
):
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]

Tool 2: edit_document

@mcp.tool(
    name="edit_document",
    description="Edit a document by replacing a string in the documents content with a new string",
)
def edit_document(
    doc_id: str = Field(description="Id of the document that will be edited"),
    old_str: str = Field(
        description="The text to replace. Must match exactly, including whitespace"
    ),
    new_str: str = Field(
        description="The new text to insert in place of the old text"
    ),
):
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    docs[doc_id] = docs[doc_id].replace(old_str, new_str)

Diem quan trong cua @mcp.tool()

1
name -- Ten tool trong JSON Schema. Claude se thay ten nay khi discovery.
2
description -- Mo ta cho Claude hieu khi nao nen dung tool nay.
3
Field(description=...) -- Mo ta tung tham so. Giup Claude hieu ro cach truyen doi so.
4
Type hints -- SDK tu dong tao JSON Schema validation. str thanh {"type": "string"}.

4. Dinh nghia Resources -- App-Controlled Primitives

Resources la du lieu ma ung dung cua ban quyet dinh khi fetch. Giong nhu GET endpoint trong HTTP server -- chi doc, khong thay doi du lieu.

Direct Resource (Static URI)

@mcp.resource("docs://documents", mime_type="application/json")
def list_docs() -> list[str]:
    return list(docs.keys())

Templated Resource (Parameterized URI)

@mcp.resource("docs://documents/{doc_id}", mime_type="text/plain")
def fetch_doc(doc_id: str) -> str:
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]

So sanh Direct vs Templated Resource

Dac diemDirect ResourceTemplated Resource
URIdocs://documents (co dinh)docs://documents/{doc_id} (co tham so)
MIME Typeapplication/jsontext/plain
Use caseDanh sach tat ca tai lieuDoc noi dung mot tai lieu cu the

MIME Types -- Kieu du lieu tra ve

MIME TypeSu dung cho
"application/json"Du lieu co cau truc (list, dict)
"text/plain"Van ban thuong
"application/pdf"File nhi phan

SDK tu dong xu ly serialization -- ban chi can return du lieu, SDK lam phan con lai.

5. Dinh nghia Prompts -- User-Controlled Primitives

Prompts la nhung instruction template nguoi dung kich hoat. Tra ve danh sach Message objects de gui truc tiep den Claude.

from mcp.server.fastmcp.prompts import base

@mcp.prompt(
    name="format",
    description="Rewrites the contents of the document in Markdown format.",
)
def format_document(
    doc_id: str = Field(description="Id of the document to format"),
) -> list[base.Message]:
    prompt = f"""
    Your goal is to reformat a document to be written with markdown syntax.

    The id of the document you need to reformat is:
    <document_id>
    {doc_id}
    </document_id>

    Add in headers, bullet points, tables, etc as necessary. Feel free to add
    in extra text, but don't change the meaning of the report.
    Use the 'edit_document' tool to edit the document. After the document has
    been edited, respond with the final version of the doc. Don't explain your changes.
    """
    return [base.UserMessage(prompt)]

Diem quan trong cua Prompts

1
Tra ve list[base.Message] -- Phai tra ve danh sach Message objects (UserMessage, AssistantMessage).
2
Tham so dong -- doc_id duoc truyen vao luc runtime, cho phep tao prompt dynamic.
3
Chat trong prompt -- Prompt co the chi dan Claude su dung tools (vi du: edit_document).

6. Chay Server -- mcp.run()

if __name__ == "__main__":
    mcp.run(transport="stdio")

transport="stdio" nghia la gi?

Server se lang nghe tren stdin/stdout -- doc JSON-RPC messages tu standard input va ghi ket qua ra standard output. Day la transport pho bien nhat khi server va client chay cung mot may.

Day la ly do ban khong the chay python mcp_server.py va tuong tac truc tiep -- server dang cho JSON-RPC messages, khong phai keyboard input.

Nghep cua mcp.run(transport="stdio") nhu bat dau mot cuoc thoai qua walkie-talkie -- server luon trong trang thai "san sang", doi tin hieu tu client va phan hoi. Stdio la kenh giao tiep, giong nhu tan so cua walkie-talkie.

    if doc_id not in docs:

Code Walkthrough -- Tim hieu Server qua tung phan

Nhan vao tung phan ben duoi de xem code chi tiet va JSON-RPC format tuong ung.

Tools -- Model-Controlled

Claude tu quyet dinh khi goi tools. Moi tool la mot Python function voi decorator @mcp.tool().

read_doc_contents

@mcp.tool(name="read_doc_contents",
          description="Read the contents of a document and return it as a string.")
def read_document(doc_id: str = Field(description="Id of the document to read")):
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]
JSON-RPC Request (Client gui den Server):
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "read_doc_contents",
    "arguments": { "doc_id": "report.pdf" }
  }
}
JSON-RPC Response (Server tra ve):
{
  "jsonrpc": "2.0",
  "result": {
    "content": [
      { "type": "text", "text": "The report details the state of a 20m condenser tower." }
    ]
  }
}

Server Architecture -- 3 Primitives
Client --tools/call--> @mcp.tool() --> docs dict
Client --resources/read--> @mcp.resource() --> docs dict
