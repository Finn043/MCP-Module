1. Roots giải quyết vấn đề gì?

Imagine bạn có MCP server với tool convert video. User nói "convert biking.mp4" nhưng chỉ có filename, không có full path. Claude không biết tìm file ở đâu trong hệ thống file phức tạp của user.

Roots giải quyết vấn đề này bằng cách đóng vai trò hệ thống permission + discovery mechanism:

  • Permission: Chỉ cho phép server truy cập thư mục được approve
  • Discovery: Claude biết tìm file ở đâu — chỉ trong approved directories
Roots giống như bạn cho thợ sửa ống nước khóa của căn hộ mình, nhưng chỉ cho phòng bếp và phòng tắm. Thợ có thể tự do hoạt động trong hai phòng đó nhưng không thể vào phòng ngủ hay phòng làm việc. Đồng thời thợ biết chỉ cần tìm ở bếp và toilet, không cần lục toàn bộ nhà.

2. Roots Workflow

Khi roots được cấu hình, flow hoạt động như sau:

User: "convert biking.mp4"
↓
Claude → list_roots → Client trả về paths
↓
Claude tìm trong roots → Tìm thấy file!
↓
convert_video("/Users/me/Desktop/biking.mp4")

3. Server-side Implementation

Server dùng ctx.session.list_roots() để discover accessible directories. Pattern phổ biến là tạo helper is_path_allowed() dùng relative_to():

# mcp_server.py
from pathlib import Path
from mcp.server.fastmcp import FastMCP, Context
from pydantic import Field

mcp = FastMCP("VidsMCP", log_level="ERROR")

async def is_path_allowed(requested_path: Path, ctx: Context) -> bool:
    """Check if path falls within approved roots."""
    roots_result = await ctx.session.list_roots()
    client_roots = roots_result.roots

    if not requested_path.exists():
        return False

    if requested_path.is_file():
        requested_path = requested_path.parent

    for root in client_roots:
        root_path = file_url_to_path(root.uri)
        try:
            requested_path.relative_to(root_path)
            return True
        except ValueError:
            continue

    return False

@mcp.tool()
async def convert_video(
    input_path: str = Field(description="Path to the input MP4 file"),
    format: str = Field(description="Output format (e.g. 'mov')"),
    *, ctx: Context,
):
    """Convert an MP4 video to another format using ffmpeg"""
    input_file = VideoConverter.validate_input(input_path)

    if not await is_path_allowed(input_file, ctx):
        raise ValueError(f"Access to path is not allowed: {input_path}")

    return await VideoConverter.convert(input_path, format)

@mcp.tool()
async def read_dir(
    path: str = Field(description="Path to a directory to read"),
    *, ctx: Context,
):
    """Read dir contents. Path must be within roots."""
    requested_path = Path(path).resolve()
    if not await is_path_allowed(requested_path, ctx):
        raise ValueError("Error: can only read directories within a root")
    return [entry.name for entry in requested_path.iterdir()]

4. Client-side Implementation

Client convert paths thành Root objects với file:// URIs và cung cấp callback:

# mcp_client.py
from mcp.types import Root, ListRootsResult, ErrorData
from mcp.shared.context import RequestContext
from pydantic import FileUrl

class MCPClient:
    def __init__(self, command, args, env=None, roots=None):
        self._roots = self._create_roots(roots) if roots else []

    def _create_roots(self, root_paths: list[str]) -> list[Root]:
        """Convert path strings to Root objects."""
        roots = []
        for path in root_paths:
            p = Path(path).resolve()
            file_url = FileUrl(f"file://{p}")
            roots.append(Root(uri=file_url, name=p.name or "Root"))
        return roots

    async def _handle_list_roots(
        self, context: RequestContext["ClientSession", None]
    ) -> ListRootsResult | ErrorData:
        """Callback for when server requests roots."""
        return ListRootsResult(roots=self._roots)

    async def connect(self):
        self._session = await self._exit_stack.enter_async_context(
            ClientSession(
                _stdio, _write,
                list_roots_callback=self._handle_list_roots
                    if self._roots else None,
            )
        )
Cách chạy: Roots được cung cấp qua CLI arguments:
uv run main.py /Users/me/Desktop /Users/me/Movies

5. MCP SDK KHÔNG tự động enforce restrictions

⚠️ CẢNH BÁO BẢO MẬT: MCP SDK không tự động giới hạn truy cập file. Bạn PHẢI tự implement is_path_allowed() và gọi nó trong mọi tool access file. Nếu không, server có thể truy cập bất kỳ file nào trên hệ thống.

Pattern đúng: luôn check is_path_allowed() trước khi thực hiện bất kỳ file operation nào.

6. Security

Roots chỉ giới hạn directories được approve. Nếu chỉ cấp Desktop, server không thể access Documents:

Lợi ích bảo mật:
  • Users chỉ cấp quyền truy cập cần thiết
  • Server không thể đọc sensitive files ngoài approved areas
  • Clear boundary giữa zones được phép và không được phép
  • Flexibility — có thể cấp/ thu hồi bất kỳ lúc nào
