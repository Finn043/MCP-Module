1. Tools là gì?

Tools là primitive được điều khiển bởi model (model-controlled). Điều này có nghĩa là Claude tự quyết định khi nào gọi tool — không phải ứng dụng của bạn, cũng không phải người dùng.

Đặc điểm chính của Tools:

  • Được điều khiển bởi Claude (AI model)
  • Claude tự động quyết định khi nào cần gọi tool dựa trên ngữ cảnh
  • Là primitive phổ biến nhất trong MCP
  • Cho phép Claude thực hiện hành động: đọc file, gọi API, tính toán, chỉnh sửa dữ liệu...
Hãy tưởng tượng Claude là một quản lý dự án. Tools chính là các "công cụ" trên bàn làm việc — bút, giấy, máy tính. Claude tự biết khi nào cần dùng công cụ nào để hoàn thành công việc.

2. Định nghĩa Tool với Python SDK

Python MCP SDK sử dụng decorator và type hints để định nghĩa tools — không cần viết JSON schema thủ công.

from mcp.server.fastmcp import FastMCP
from pydantic import Field

mcp = FastMCP("DocumentMCP")

docs = {
    "deposition.md": "This deposition covers the testimony of Angela Smith, P.E.",
    "report.pdf": "The report details the state of a 20m condenser tower.",
    "financials.docx": "These financials outline the project's budget and expenditures.",
}

@mcp.tool(
    name="read_doc_contents",
    description="Read the contents of a document and return it as a string.",
)
def read_document(
    doc_id: str = Field(description="Id of the document to read"),
):
    """Đọc nội dung tài liệu theo ID."""
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]

@mcp.tool(
    name="edit_document",
    description="Edit a document by replacing a string in the document.",
)
def edit_document(
    doc_id: str = Field(description="Id of the document to edit"),
    old_str: str = Field(description="The text to replace"),
    new_str: str = Field(description="The new text to insert"),
):
    """Chỉnh sửa tài liệu bằng cách thay thế text."""
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    docs[doc_id] = docs[doc_id].replace(old_str, new_str)

Cách SDK hoạt động:

  • @mcp.tool() decorator đăng ký function tự động
  • name — tên tool (hiển thị cho Claude)
  • description — mô tả tool (giúp Claude hiểu chức năng)
  • Field() từ pydantic — mô tả từng parameter
  • Type hints (str) tự động sinh JSON schema

3. tools/list Request/Response

Khi client kết nối đến MCP server, nó gửi request tools/list để khám phá tất cả tools có sẵn.

Request (Client → Server):

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list"
}

Response (Server → Client):

{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [
      {
        "name": "read_doc_contents",
        "description": "Read the contents of a document and return it as a string.",
        "inputSchema": {
          "type": "object",
          "properties": {
            "doc_id": {
              "type": "string",
              "description": "Id of the document to read"
            }
          },
          "required": ["doc_id"]
        }
      },
      {
        "name": "edit_document",
        "description": "Edit a document by replacing a string in the document.",
        "inputSchema": {
          "type": "object",
          "properties": {
            "doc_id": {"type": "string", "description": "Id of the document to edit"},
            "old_str": {"type": "string", "description": "The text to replace"},
            "new_str": {"type": "string", "description": "The new text to insert"}
          },
          "required": ["doc_id", "old_str", "new_str"]
        }
      }
    ]
  }
}

Server trả về danh sách tools với name, description, và inputSchema — tất cả được tự động sinh từ Python code.

4. tools/call Request/Response

Khi Claude quyết định cần dùng tool, client gửi tools/call với tên tool và arguments.

Request (Client → Server):

{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "read_doc_contents",
    "arguments": {
      "doc_id": "report.pdf"
    }
  }
}

Response (Server → Client):

{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "The report details the state of a 20m condenser tower."
      }
    ],
    "isError": false
  }
}

5. Error Handling trong Tools

Khi tool gặp lỗi, MCP xử lý tự động thông qua isError flag.

Ví dụ: gọi tool với doc_id không tồn tại

# Python function raise ValueError
def read_document(doc_id: str):
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]

JSON Response khi có lỗi:

{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Error: Doc with id nonexistent.pdf not found"
      }
    ],
    "isError": true
  }
}

Lưu ý "isError": true — client và Claude biết rằng kết quả là lỗi, không phải dữ liệu hợp lệ.

6. Claude's Tool Use Loop

Đây là quy trình agentic loop — vòng lặp mà Claude sử dụng tools để hoàn thành task.

User Query → Gửi query + tools cho Claude → Claude phản hồi

↓ Nếu stop_reason = "tool_use"

Thực thi tool → Feed result lại cho Claude → Claude phản hồi tiếp

↓ Lặp lại cho đến khi stop_reason = "end_turn"

Trả về kết quả cuối cùng cho User
1
Gửi query + tools — Ứng dụng gửi câu hỏi của user kèm danh sách tools cho Claude.
2
Claude quyết định — Nếu cần tool, stop_reason sẽ là "tool_use". Nếu không, trả về text trực tiếp.
3
Thực thi tool — Client gọi tools/call trên MCP server với arguments mà Claude cung cấp.
4
Feed result back — Kết quả tool được gửi lại cho Claude trong conversation tiếp theo.
5
Lặp lại — Claude có thể gọi thêm tools hoặc trả về kết quả cuối cùng (stop_reason = "end_turn").
