Hiểu cách process truy cập stdin, stdout, files — nền tảng của MCP transport

1. File Descriptor = Số đại diện cho "mở file"

Trong Unix, khi process mở một file (hoặc stream), OS trả về một số nguyên — đó là File Descriptor (FD).

Process mở file → OS cấp FD

open("data.txt") ──→ FD = 3

FD chỉ là số, nhưng OS dùng nó để biết "file nào đang được mở"

2. Ba FD mặc định — Tự động có khi process khởi động

FDTênHướngMặc định trỏ đến
0stdinREADBàn phím (keyboard)
1stdoutWRITETerminal (màn hình)
2stderrWRITETerminal (màn hình)
Key insight: stdin, stdout, stderr cũng là files. Mọi thứ trong Unix đều là file — bàn phím, màn hình, socket, pipe đều là file.

3. FD Table trong mỗi Process

Process (python3)

│ FD Table │
├─ FD 0 ──→ stdin (keyboard)
├─ FD 1 ──→ stdout (terminal)
├─ FD 2 ──→ stderr (terminal)
├─ FD 3 ──→ /home/user/data.txt
└─ FD 4 ──→ socket(127.0.0.1:8080)

4. Redirect — Thay đổi nơi FD trỏ đến

Shell cho phép redirect FD sang nơi khác trước khi chạy lệnh:

# Redirect stdout (FD 1) vào file
$ echo "Hello" > output.txt

# Redirect stdin (FD 0) từ file
$ python3 server.py < input.json

# Redirect stderr (FD 2) vào file riêng
$ python3 server.py 2> debug.log

# Redirect stderr vào stdout
$ python3 server.py 2>&1

# Cả hai vào file khác nhau
$ python3 server.py > out.txt 2> err.log

5. Pipe = Nối FD 1 của process A với FD 0 của process B

Process A stdout (FD 1) ──pipe──→ stdin (FD 0) Process B

Lệnh: python3 producer.py | python3 consumer.py

6. FD trong Python

import sys

# Xem số FD của standard streams
print(sys.stdin.fileno())   # → 0
print(sys.stdout.fileno())  # → 1
print(sys.stderr.fileno())  # → 2

# Mở file → nhận FD mới
with open("data.txt", "r") as f:
    print(f.fileno())  # → 3 (hoặc số cao hơn)

7. Tại sao FD quan trọng cho MCP?

MCP stdio transport:
• MCP Host spawn MCP Server process
• Host redirect FD 0 (stdin) của Server ← pipe từ Host
• Host redirect FD 1 (stdout) của Server → pipe về Host
• Kết quả: 2 processes giao tiếp qua FD, không cần HTTP!
