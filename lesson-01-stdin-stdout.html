1. Ba standard streams của mọi process

Mỗi process trong Unix/Linux/macOS tự động có 3 kênh giao tiếp:

StreamFDHướngMặc định kết nối đếnDùng để
📥 stdin0INPUT (đọc)Bàn phímNhận dữ liệu vào
📤 stdout1OUTPUT (ghi)TerminalGửi kết quả ra
⚠️ stderr2OUTPUT (ghi)TerminalGửi lỗi/debug
Bàn phím ──→ stdin (FD 0) ──→ Process ──→ stdout (FD 1) ──→ Terminal

stderr (FD 2) cũng ghi ra Terminal, nhưng tách biệt với stdout
Process như một nhà máy: stdin là cửa nhập nguyên liệu, stdout là cửa xuất sản phẩm, stderr là chuông báo lỗi.

2. Đọc stdin trong Python

import sys

# Đọc một dòng từ stdin (BLOCKING — chờ đến khi có data)
line = sys.stdin.readline()
print(f"Nhận được: {line.strip()}")

# Đọc toàn bộ stdin cho đến EOF
data = sys.stdin.read()

# Vòng lặp đọc từng dòng (pattern của MCP server)
while True:
    line = sys.stdin.readline()
    if not line:  # EOF — client đóng kết nối
        break
    handle(line.strip())

3. Ghi stdout trong Python

import sys

# Cách 1: print (tự thêm newline)
print("Hello, stdout!")

# Cách 2: sys.stdout.write (kiểm soát hơn)
sys.stdout.write("Hello!\n")

# ⚠️ QUAN TRỌNG: flush() sau khi ghi trong MCP!
sys.stdout.flush()  # Đảm bảo data được gửi ngay, không bị buffer
⚠️ Tại sao cần flush()?
Python buffer stdout để tối ưu performance. Trong MCP, nếu không flush, message có thể bị giữ lại trong buffer — client sẽ không nhận được response!

4. stderr — Kênh riêng cho lỗi và debug

import sys

# Ghi ra stderr — KHÔNG bị redirect cùng stdout
print("Debug info", file=sys.stderr)
sys.stderr.write("Error: something failed\n")

# Redirect riêng biệt trong shell:
# $ python3 server.py > output.txt 2> error.log
💡 Trong MCP: Dùng stderr cho debug logs.
MCP host chỉ đọc stdout — stderr không ảnh hưởng giao tiếp.

5. Pipe — Kết nối stdout → stdin

Ký tự | kết nối stdout của lệnh trước với stdin của lệnh sau:

# stdout của echo → stdin của python3
$ echo '{"method":"ping"}' | python3 mcp_server.py

# Chuỗi nhiều pipe
$ cat requests.txt | python3 server.py | python3 parser.py

6. Pattern MCP Server hoàn chỉnh

import sys, json

while True:
    line = sys.stdin.readline()     # ← Nhận JSON-RPC từ host
    if not line: break

    request = json.loads(line)
    response = process(request)

    sys.stdout.write(json.dumps(response) + "\n")
    sys.stdout.flush()              # ← Gửi ngay, không buffer

Bài tập 1: Đọc từ stdin ⬜ Chưa làm

Viết code để đọc một dòng từ stdin và in ra màn hình. Code của bạn sẽ được kiểm tra cú pháp và cấu trúc.

🐍 Python Editor
📤 Output
💡 Gợi ý: Dùng sys.stdin.readline() để đọc dòng, rồi dùng print() để in ra. Đừng quên .strip() để xóa newline!
Giải pháp:
import sys line = sys.stdin.readline() print(f"Đọc được: {line.strip()}")
Bài tập 2: Ghi stdout với flush() ⬜ Chưa làm

Viết code để ghi message ra stdout và đảm bảo nó được gửi ngay lập tức (không bị buffer).

🐍 Python Editor
📤 Output
💡 Gợi ý: Dùng sys.stdout.write(message) để ghi, rồi gọi sys.stdout.flush() để ép gửi ngay.
Giải pháp:
import sys sys.stdout.write("Hello MCP!\n") sys.stdout.flush()
